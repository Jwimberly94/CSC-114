Question 1: Loss Function for Regression
According to the Module 3 reading "How Machine Learning Works," 
the recommended loss function for regression problems is Mean Squared Error (MSE).

The reading explains the "why" clearly: regression problems are those where you predict a number — 
like a house sale price. MSE measures how wrong each prediction was by squaring each miss before averaging 
them together.

The reading notes something important about that squaring: it means a few enormous misses hurt the score far 
more than many small ones. This pushes the model to avoid catastrophically bad predictions, 
not just average ones.

The reading contrasts this with classification problems (predicting a category, 
like spam/not spam), which use cross-entropy instead.

Question 2: The Five Steps of the Training Loop
From the same reading, in order:

Feed data — a batch of examples enters the model
Predict — the model outputs a guess
Compute loss — one number measuring how wrong the guess was
Gradient — determine which way to adjust each parameter
Update parameters — the optimizer nudges every number
Then it loops back to Step 1 and repeats — thousands of times.

Question 3: Positive Gradient = Increase the Parameter?
Negative, Captain. That is backwards.

The Module 3 reading states this explicitly in its gradient table:

Gradient Value	What it means	So you should…
Positive (+)	loss goes up if this parameter goes up	DECREASE this parameter
Negative (−)	loss goes up if this parameter goes down	INCREASE this parameter
The logic: you always want to move in the direction that lowers the loss. A positive gradient means going 
up makes things worse, so you go the opposite direction — down.

This is literally what gradient descent means. You move against the gradient.

Question 4: Zero Loss is the Normal Goal?
Also negative, Captain.

The reading is direct on this point:

"Zero loss → perfect (basically never happens in real life)"

Zero loss would mean every single prediction was perfect. Real data is messy. The actual goal is to 
reduce the loss as much as possible — not reach zero.
